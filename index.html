About Project

File System Emulator + Page Replacement Analyzer — A dual-purpose academic tool that mimics a mini operating system's file manager with hierarchical directory structure and CLI simulation, while also providing an in-depth analysis of virtual memory page replacement algorithms: FIFO, LRU, and Optimal.


The File System Emulator concept demonstrates core file operations (create, delete, read, write) and directory management using a tree structure. Meanwhile, the page replacement simulation helps understand how OS decides which memory pages to swap, influencing system efficiency and page fault rate.

This project bridges file management and memory management — two pillars of OS fundamentals.

Page Replacement Algorithms

🔄 FIFO (First-In-First-Out)

Explanation: Oldest page in memory is replaced first. Simple queue structure.

Advantage: Easy to implement. Disadvantage: Belady's anomaly (more frames can increase faults).

⏳ LRU (Least Recently Used)

Explanation: Replaces the page that hasn't been used for the longest time.

Advantage: Better performance, reflects temporal locality. Disadvantage: Hardware support needed, costly.

🎯 Optimal Page Replacement

Explanation: Replaces page that will not be used for the longest future. Theoretical optimum.

Advantage: Lowest page faults. Disadvantage: Impossible to implement in practice (requires future knowledge).
